# Multi-Word Phrase Combination — Design

## Problem

The matching engine only returns pre-indexed entries (single words from CMU dict + corpus phrases). For phonemes that rarely end long words (AW, OY, AA, etc.), there are few or zero single-word matches at higher syllable counts. Example: zero 4-syllable words end in AW. The good rhymes for "body verb now" are multi-word phrases like "got the words down" — but the engine can't generate those.

## Solution

Add runtime word combination to matching.py. The engine dynamically builds 2-word and 3-word phrases by splitting the target fingerprint and finding words that fill each piece.

User also maintains curated multi-word phrases in corpus.txt for known high-quality rhymes.

## User-Facing Behavior

- Multi-word combos appear in the same ranked list as single-word results
- Scored by the same tier system (True Match, Strong, Slant, Accent Variant)
- Single words get a slight tiebreak advantage over multi-word combos at the same score
- Corpus phrases keep their corpus bonus and star badge
- No new UI components needed

## Combination Algorithm

### Input

Target fingerprint extracted from user's phrase. Example: "body verb now" → `[AA, IY, ER, AW]` (4 syllables).

### 2-Word Combos

For each split point s (1 to N-1 where N = syllable count):
- prefix_fp = target_fp[:s]
- suffix_fp = target_fp[s:]
- Find words whose fingerprint has the right syllable count
- Last word must end with the target's final phoneme
- Score the concatenated fingerprint against the target using existing `score_entry()`

Example splits for 4 syllables:
- Split 3|1: 3-syllable prefix + 1-syllable suffix ending in AW
- Split 2|2: 2-syllable prefix + 2-syllable suffix ending in AW
- Split 1|3: 1-syllable prefix + 3-syllable suffix ending in AW

### 3-Word Combos

For each pair of split points (s1, s2) where s1 < s2:
- part1_fp = target_fp[:s1]
- part2_fp = target_fp[s1:s2]
- part3_fp = target_fp[s2:]
- Last word must end with the target's final phoneme
- Score concatenated fingerprint against target

### Scoring

- Reuses existing `score_entry()` — concatenated fingerprint vs target fingerprint
- Same tier classification: all positions match = Tier 1, 60%+ interior = Tier 2, etc.
- Word frequency weighting: combos of common words score slightly higher than combos of obscure words (uses `wordfreq` library already in project)
- Multi-word combos get a small tiebreak penalty vs single words at the same score (e.g., -1 point) so single words rank first when scores are equal

### Quality and Performance Guards

- Cap at ~200 best multi-word combos before merging with single-word results
- Only the last word in a combo needs to end with the target's final phoneme (pre-filter)
- Deduplicate: if a multi-word combo matches an existing corpus entry, skip it

## Data Changes

### New File: `data/combo_index.json`

Generated by `generate_wordlist.py`. Contains ALL words (including single-syllable) with fingerprints, indexed for fast lookup:

```json
{
  "by_syl_and_final": {
    "1": {
      "AW": [
        {"word": "down", "fp": ["AW"], "freq": 0.001},
        {"word": "now", "fp": ["AW"], "freq": 0.002}
      ],
      "AO R": [
        {"word": "door", "fp": ["AO R"], "freq": 0.0005}
      ]
    },
    "2": { ... },
    "3": { ... }
  },
  "by_syl": {
    "1": [
      {"word": "got", "fp": ["AA"], "freq": 0.001},
      ...
    ],
    "2": [ ... ],
    "3": [ ... ]
  }
}
```

- `by_syl_and_final`: for quickly finding suffix/anchor words (last word in combo)
- `by_syl`: for finding prefix/middle words by syllable count
- Estimated size: ~5-8MB
- Committed to repo alongside wordlist.json

### Modified Files

| File | Change |
|------|--------|
| `scripts/generate_wordlist.py` | Also outputs `data/combo_index.json` |
| `scripts/matching.py` | New `generate_combos()` function; results merged into main output |

### Unchanged

- API route (`app/api/rhyme/route.ts`) — no changes needed
- All frontend components — no changes needed
- Scoring logic (`score_entry`, `apply_corpus_bonus`) — reused as-is
- `data/wordlist.json`, `data/corpus.json`, `data/phoneme_lookup.json` — unchanged

## Performance

- combo_index.json load: ~100ms
- Single-word matching: ~0.5-1s (unchanged)
- Multi-word combination search: ~1-2s
- Total estimated query time: ~2-3s (up from ~1s)
- Vercel free tier limit: 10s — well within budget

## Corpus Expansion (Parallel Effort)

User adds known high-quality multi-word rhymes to `data/corpus.txt`. These get preprocessed into `corpus.json` as before and receive the corpus bonus (+25, capped). Runtime combination fills gaps; corpus provides curated quality.
